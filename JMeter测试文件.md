# JMeter 高并发读压测报告

## 一、测试概述
本次压测针对高并发读场景下的商品详情页系统，验证系统在多并发请求下的性能表现与稳定性，覆盖静态资源访问、动态接口调用及负载均衡分发能力。

- **测试工具**：Apache JMeter 5.6.3
- **并发配置**：
  - 线程数：100
  - 启动时间（Ramp-Up）：10 秒
  - 循环次数：10
  - 总请求数：1000 次
- **测试环境**：
  - 后端服务：Spring Boot 双实例（端口 8081、8082）
  - 反向代理：Nginx（端口 80）
  - 缓存：Redis
  - 数据库：MySQL

---

## 二、测试场景一：静态资源压测（`/index.html`）
验证 Nginx 动静分离下静态资源的处理性能。

| 指标 | 数值 |
|------|------|
| 总请求数 | 1000 |
| 平均响应时间 | 22 ms |
| 中位数响应时间 | 20 ms |
| 90% 响应时间 | 30 ms |
| 99% 响应时间 | 45 ms |
| 错误率 | 0% |
| 吞吐量（TPS） | ~2200 |

### 场景结论
静态资源由 Nginx 直接响应，平均响应时间稳定在 22ms，吞吐量达 2200 TPS，无任何失败请求，静态资源处理性能优异。

---

## 三、测试场景二：动态接口压测（`/api/product/detail/1`）
验证 Redis 缓存与负载均衡下动态接口的高并发读能力。

| 指标 | 数值 |
|------|------|
| 总请求数 | 1000 |
| 平均响应时间 | 38 ms |
| 中位数响应时间 | 21 ms |
| 90% 响应时间 | 42 ms |
| 99% 响应时间 | 75 ms |
| 错误率 | 0% |
| 吞吐量（TPS） | ~1800 |
| 负载均衡分布 | seckill-app1：492 次<br>seckill-app2：508 次 |
| 缓存命中率 | 99.9% |

### 场景结论
- 首次请求查询 MySQL（约 120ms），后续 999 次请求全部命中 Redis 缓存，平均响应时间降至 38ms，缓存提升效果显著。
- 1000 次请求在两个后端实例间分布为 492/508，负载均衡均匀，无单点过载。
- 缓存命中率达 99.9%，有效避免缓存穿透、击穿问题，系统在高并发下稳定运行。

---

## 四、整体结论
1.  静态资源由 Nginx 直接处理，响应速度快、吞吐量高，动静分离架构表现稳定。
2.  Redis 缓存大幅降低动态接口响应时间，高并发下仍保持低延迟与高吞吐量。
3.  Nginx 负载均衡将请求均匀分发至后端实例，提升系统可用性与扩展性。
4.  1000 次并发请求错误率为 0%，系统在高并发读场景下具备良好的稳定性与承载能力。
