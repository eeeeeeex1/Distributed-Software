# 商务秒杀系统容器部署配置说明

|----------|------|----------|----------|
| seckill-mysql | 运行中 | 3307:3306 | ✅ healthy |
| seckill-redis | 运行中 | 6379:6379 | ✅ healthy |
| seckill-service | 运行中 | 8080:8080 | ✅ healthy |
| seckill-nginx | 运行中 | 80:80 | ✅ healthy |

## 配置详情

### 1. 独立容器实例
- MySQL 8.0 : 数据库服务
- Redis 7 : 缓存服务
- Spring Boot : 后端服务
- Nginx Alpine : 反向代理服务

### 2. 网络通信
- 所有容器连接到 seckill-network 桥接网络
- 容器间通过服务名称互相访问

### 3. 资源限制
| 服务 | CPU 限制 | 内存限制 |
|------|----------|----------|
| MySQL | 1.0 核 | 1GB |
| Redis | 0.5 核 | 512MB |
| 后端服务 | 2.0 核 | 1GB |
| Nginx | 0.5 核 | 256MB |

### 4. 自动重启策略
- 所有容器配置 restart: always

### 5. 数据持久化
- mysql-data : MySQL 数据卷
- redis-data : Redis 数据卷
- nginx-logs : Nginx 日志卷

### 6. 健康检查
- MySQL : mysqladmin ping (10s 间隔)
- Redis : redis-cli ping (10s 间隔)
- 后端服务 : /actuator/health (30s 间隔)
- Nginx : nginx -t (30s 间隔)

### 7. 启动顺序

MySQL → Redis → 后端服务 → Nginx
通过 depends_on 和 healthcheck 条件确保依赖关系

### 8. 日志配置
- 驱动: json-file
- 最大文件大小: 20-100MB
- 最大文件数: 3-5 个

## 访问地址
- 前端页面 : http://localhost
- API 接口 : http://localhost/api/
- 后端直连 : http://localhost:8080
- 健康检查 : http://localhost:8080/actuator/health
